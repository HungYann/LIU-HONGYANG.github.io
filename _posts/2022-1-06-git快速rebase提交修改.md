### 问题起源

在和同事配合开发的时，经常会遇到如下问题，`自己开启一个分支开发后，开发完成，正准备提交的时候，别人在你之前，已经提交`，

**此时，如果你再次进行提交，就会出现冲突。**

**那么如何解决呢？**

首先说自己的传统方法:

- 传统的办法是:

通过`git reset`复原**master**到某个历史提交点，然后pull拉取最新的提交记录，
接着将刚刚开发的分支上的记录rebase到master最新节点上。

这样master节点就会包含他人的提交点和自己最新的提交。

- 修正办法是:

在当前的提交点上，使用

```
git pull origin master --rebase
```
命令

这样可以将对方的代码合并过来。
